## 문제 상황

사내 PC 보안 정책

MIDAS 폴더 외부 파일 → 자동 암호화 적용

## 기존 플로우 (문제 발생)

```
┌──────────────────┐
│ 대용량 PDF 파일  │
│ (설계 문서 등)   │
└────────┬─────────┘
         │
         ▼
┌──────────────────────────────────────┐
│ AI Coding Tool에서 PDF 읽기 시도     │
│ - Read 기능 사용                     │
│ - PDF Reader MCP 사용                │
└────────┬─────────────────────────────┘
         │
         ▼
┌──────────────────────────────────────┐
│ 캐시 파일 생성                       │
│ 📁 C:\Users\{사용자}\.claude\projects │
│ └── cache\_xxxxx.txt                 │
└────────┬─────────────────────────────┘
         │
         ▼
┌──────────────────────────────────────┐
│ ⚠️ 보안 정책 자동 적용               │
│ → txt 파일 암호화됨                  │
└────────┬─────────────────────────────┘
         │
         ▼
┌──────────────────────────────────────┐
│ ❌ AI Coding Tool 캐시 읽기 실패     │
│ → 데이터 처리 불가                   │
│ → 작업 중단                          │
└──────────────────────────────────────┘
```

## 원인 분석

| 항목 | 내용 |
| --- | --- |
| 기본 캐시 경로 | `C:\Users\{사용자}\.claude\` |
| 보안 정책 적용 범위 | `C:\MIDAS\` 외부 모든 경로 |
| 문제 | 캐시 경로가 보안 정책 적용 범위에 포함됨 |

## 해결 방법

**환경 변수** `CLAUDE_CONFIG_DIR` 설정

시스템 환경 변수 추가

| **항목** | **내용** |
| --- | --- |
| 변수명 | CLAUDE_CONFIG_DIR |
| 값 | C:\MIDAS\.claude |

### 설정 방법

1. **Windows 설정** → **시스템** → **정보** → **고급 시스템 설정**
2. **환경 변수** 클릭
3. **사용자 변수** 또는 **시스템 변수**에서 **새로 만들기**
4. 아래 값 입력:
   * 변수 이름: `CLAUDE_CONFIG_DIR`
   * 변수 값: `C:\MIDAS\.claude` (또는 MIDAS 하위 원하는 경로)
5. **확인** 후 **터미널/IDE 재시작**
